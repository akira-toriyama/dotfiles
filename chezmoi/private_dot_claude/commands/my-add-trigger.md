あなたのタスク: `~/.claude/CLAUDE.md` の「要所トリガー索引」に **新しい1エントリ**を追加する。

## 入力
`$ARGUMENTS` に「いつ → 何を読む」が渡されていればそれを使う。無ければユーザーに**2つだけ**質問する:
1. **いつ?**（イベント/タイミング。例: 「新しいリポジトリを作成するとき」）
2. **何をする?**（読む公式 docs の名前＋URL。URL 不明ならトピック語だけでよい）

## 手順
1. **URL を確定**: URL があれば WebFetch で実在＆該当ページを確認。無ければ WebSearch（公式ドメインに絞る）で探し WebFetch で確認。**docs root でなく該当セクション / 非 Enterprise / @latest 無し**の素の URL を選ぶ。確認できなければ報告して止まる（**推測 URL を書かない**）。
2. **ドメインを判定**（GitHub / macOS / TypeScript …）。`~/.claude/CLAUDE.md` に該当 `### <ドメイン>` があればその配下、無ければ新規 `### <ドメイン>` を作る。
3. **標準フォーマットで1エントリ生成**:
   ```
   - **<いつ>** → 「<参照名>」を読む。
     <URL>
   ```
4. **本文はリンク先に置く** — docs の中身を CLAUDE.md に書かない（索引は lean に保つ）。
5. **提案 → 承認**: 挿入位置と最終エントリを見せ、**承認を得てから** `~/.claude/CLAUDE.md` を編集（黙って追記しない）。
6. **重複チェック**: 同じイベントが既にあれば、新規追加でなく更新を提案。

入力: $ARGUMENTS
